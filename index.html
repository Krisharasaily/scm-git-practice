SCM Git Practice

Welcome to the Source Control Management (SCM) Git practice page!

1. Setting Up Git

Before you can start using Git, you need to configure your username and email address:

git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"

2. Creating a New Git Repository

To initialize a new Git repository in your project folder, use:

git init

3. Cloning a Repository

If you want to clone an existing repository from GitHub or another remote repository, use:

git clone https://github.com/your-username/repository-name.git

4. Staging and Committing Changes

To add files to the staging area and commit changes:

git add .
git commit -m "Your commit message"

5. Pushing Changes to Remote

After committing your changes locally, push them to the remote repository:

git push origin main

6. Pulling Changes from Remote

To fetch and merge changes from the remote repository to your local machine:

git pull origin main

7. Viewing Commit History

To view the commit history of the repository:

git log

Enjoy practicing Git SCM!
